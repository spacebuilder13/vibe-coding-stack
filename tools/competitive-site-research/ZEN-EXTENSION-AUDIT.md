# Competitive Research Skill — ZEN Extension Audit

## Scope

Audit of the **skill workflow itself** against ZEN principles after extension.

## Working

- **Clarity First**: skill now has explicit ZEN output artifacts, not implicit expectations.
- **Progressive Disclosure**: core crawl flow remains simple; ZEN audit is an additive step.
- **Trust by Transparency**: score semantics (`PASS/PARTIAL/FAIL`) and confidence model are documented.
- **Actionable Outcomes**: findings doc requires prioritized remediation list.

## Not working / partial

- **Low Cognitive Load (partial)**: ZEN extension currently lives across multiple docs; newcomers may need a single quickstart checklist.
- **Consistency (partial)**: prompts and skill docs now both mention audit outputs, but templates are not yet auto-generated by script.
- **Accessibility (partial)**: workflow does not yet enforce DOM/ARIA checks; only content-level evidence is guaranteed.

## Logic for next extension iteration

1. Add `scripts/init_zen_audit_docs.py` to scaffold rubric, scorecard, findings.
2. Add optional `--zen-audit` flag in crawler CLI to generate skeleton docs after crawl.
3. Add screenshot capture checklist for accessibility evidence (focus order, labels, contrast notes).
